### 修改总结

问题根因：行高（22px）、字节格宽（27px）、字符格宽（11px）、偏移列宽（90px）全部是硬编码固定值，字号变大时文字内容已超出容器。

**解决方案：以 `fontSize=13` 为基准，所有尺寸按比例缩放。**

#### HexEditor.vue

- 新增 5 个 computed 尺寸变量（基于 `settingsStore.fontSize`）：
  - `rowH = fontSize × 22/13`（数据行高）
  - `tagH = fontSize × 16/13`（标签区高度）
  - `byteCellW = fontSize × 27/13`（字节格宽）
  - `charCellW = fontSize × 11/13`（解码字符格宽）
  - `offsetColW = fontSize × 90/13`（偏移列宽）
- `ROW_HEIGHT` 从常量 `38` 改为 `computed(() => rowH + tagH)`，虚拟滚动计算自动跟随字号变化
- 在 `.hex-editor__content` 的 `:style` 中注入 5 个 CSS 变量（`--hex-row-h` 等），传递给所有子组件
- 列头 `font-size: 12px` → `font-size: inherit`，偏移列宽/字节标签格宽改为 `var(...)` CSS 变量

#### HexGrid.vue

- 新增 `byteCellW`、`charCellW` computed（同公式），用于模板绑定和标签定位 JS 计算
- CSS 中所有硬编码 px 值全部改为 `var(--hex-row-h)`、`var(--hex-byte-w)` 等 CSS 变量
- 标签字号改为 `0.85em`（相对单位，随父字号缩放）
