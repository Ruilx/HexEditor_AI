# HexEditor 项目状态与设计快照 (2026-04-29)

## 1. 项目基础架构状态
* **技术栈**：已成功使用 Vue3 + Vite 作为基础框架，状态管理使用 Pinia，UI 组件库接入了 ant-design-vue。符合项目的选型要求。
* **文件结构**：目录结构已清晰划分，包含 `components` (UI组件)、`core` (核心逻辑如 FileBuffer/EditHistory等)、`plugins` (显示信息插件框架)、`stores` (全局状态)、`utils` (工具函数，如fp8/fp16/encoding)。

## 2. 已实现的功能和框架
* **界面结构搭建**：`App.vue` 中已完成核心布局，包括菜单栏（MenuBar）、侧边栏（AppSidebar）、标签栏（TabBar）、编辑器主体（HexEditor）和状态栏（StatusBar）。
* **虚拟滚动渲染**：`HexEditor.vue` 中已经实现了针对大文件的虚拟滚动机制（Virtual Scrolling），通过计算可见行和预加载行（OVERSCAN），解决了大文本的渲染卡顿问题，实现了“分块加载显示”理念。
* **状态管理 (Stores)**：
  * `fileStore.js`: 文件管理与切换。
  * `settingsStore.js`: 用户偏好设置（如每行字节数）。
  * `editorStore.js`: 编辑器核心交互（选中、复制、粘贴等快捷命令）。
  * `tagStore.js`: 标签状态管理。
* **交互框架**：
  * **右键菜单框架** (`ContextMenu.vue`)：实现了根据鼠标是否在选中区域弹出不同类型菜单的支持，涵盖了剪切、复制、粘贴、偏移量复制、标签管理、设置文件大小等子菜单。
  * **对话框集成**：集成了查找替换 (`FindReplace.vue`)、标签编辑器 (`TagEditor.vue`)、设置对话框 (`SettingsDialog.vue`) 等骨架。

## 3. 待完善与待实现功能
尽管框架已经搭建，仍有大量核心业务逻辑需要填充：
* **二进制编辑核心逻辑**：
  * 插入与替换模式的具体实现。
  * `FileBuffer.js` 中的底层二进制流修改与撤销/重做记录 (`EditHistory.js`)。
  * 文件保存（导出为本地二进制文件）。
* **数据解读面板 (DataInfoPanel)**：
  * 支持项目中所述的所有数据格式解析（有/无符号整型、FP8/FP16等各种精度的浮点数、GUID等）。
  * 需要在 `plugins/interpreters/` 下完善对应的解析实现逻辑，并支持大小端切换。
* **文本解码面板**：
  * 按需切换 UTF-8, GBK, SHIFT-JIS 等编码在视图右侧正确显示对应的字符内容，并做高亮联动。
* **高级检索与替换**：
  * 真正的十六进制级别查找，并支持批量替换功能。
* **标签系统 (.TAG)**：
  * 完整的标签渲染器（在十六进制视图间预留空隙并绘制背景），并且实现拖拽绑定、源文件读取时同目录自动读取 `.TAG` 文件等功能。
* **其他特色功能**：计算 Hash/校验和与 Base 编码的延迟计算逻辑。

## 4. 后续方案和计划
1. **完善底层基建**：优先完成 `FileBuffer` 及 `EditHistory` 操作，跑通单字节的修改、插入、保存下载流程。
2. **实现插件体系**：将数值转换抽取到 `PluginManager`，以此支持十进制、浮点数以及字符串的实时预览显示。
3. **标签视图开发**：在虚拟列表中注入空白行用于渲染 Tag 块。增加标签 `.TAG` 导出的逻辑。
4. **编码和搜索加强**：引入所需编码的解码库来正确渲染文本区域。实现字节流搜索算法。
