## 二、实现内容

### 2.1 键盘输入编辑（`src/stores/editorStore.js`）

**新增状态**：
- `inputBuffer: ref({ offset, highNibble })`：半字节输入缓冲，记录当前正在输入的字节高4位
- `_commitByteFlag: ref({ offset, value, timestamp })`：触发器 ref，用于通知 HexGrid 执行写入
- `_copyFlag / _pasteFlag / _cutFlag`：剪贴板操作触发器

**新增方法**：
- `handleHexInput(char)`：处理单个十六进制字符输入（`0-9a-f`）
  - 首次输入：存储高4位到 `inputBuffer`
  - 二次输入：合并高低4位 → 提交完整字节
- `commitByte(offset, value)`：触发 `_commitByteFlag`，由 HexGrid 监听执行写入
- `clearInputBuffer()`：切换光标时清除未完成的半字节输入

**修改方法**：
- `setCursor(offset)`：调用前先执行 `clearInputBuffer()`，避免移位残留

**新增剪贴板接口**（触发器模式，实际逻辑在 HexGrid 中执行）：
- `copy()`：触发 `_copyFlag`
- `paste()`：触发 `_pasteFlag`
- `cut()`：先触发 `_copyFlag`，再触发 `_cutFlag`

### 2.2 键盘事件处理（`src/components/editor/HexGrid.vue`）

**组件修改**：
- 根 `<div>` 添加 `tabindex="0"` 使组件可聚焦
- 添加 `ref="gridRef"` 引用
- 绑定事件：`@keydown`、`@compositionstart`、`@compositionend`
- CSS 添加 `outline: none` 移除聚焦边框
- `onMounted` 时自动聚焦网格

**新增事件处理函数**：

| 函数 | 功能 |
|------|------|
| `onKeydown(event)` | 分发键盘事件：十六进制字符 → `handleHexInput`，方向键 → `handleArrowKey`，Delete/Backspace → `handleDeleteKey`，Ctrl 组合键 → 对应操作 |
| `handleArrowKey(key)` | ←→↑↓ 移动光标，支持跨行（↑↓ 步进 bytesPerRow） |
| `handleDeleteKey(key)` | Backspace 删除光标前字节并回退光标；Delete 删除光标当前字节；若有选区则删除选区 |
| `onCompositionStart/End()` | 屏蔽输入法（中文输入时 `isComposing=true`，忽略 keydown）|

**新增 `watch` 监听**：

```javascript
// 监听字节提交 → 写入文件 + 光标前进
watch(() => editorStore._commitByteFlag, (flag) => {
  const bytes = new Uint8Array([flag.value])
  const mode = editorStore.insertMode ? 'insert' : 'replace'
  fileStore.writeBytes(flag.offset, bytes, mode)
  editorStore.setCursor(flag.offset + 1)  // 自动前进光标
}, { deep: true })

// 监听复制 → 读取选区字节 → 写入剪贴板（十六进制格式，空格分隔）
watch(() => editorStore._copyFlag, async () => { ... })

// 监听粘贴 → 读取剪贴板 → 解析十六进制 → 写入文件
watch(() => editorStore._pasteFlag, async () => { ... })

// 监听剪切 → 删除选区字节
watch(() => editorStore._cutFlag, (flag) => { ... }, { deep: true })
```

**快捷键支持**（在 `onKeydown` 中处理）：

| 快捷键 | 动作 |
|--------|------|
| `0-9`, `a-f` | 十六进制字节输入（两键一字节） |
| `←` `→` `↑` `↓` | 光标移动 |
| `Backspace` | 删除前一字节 / 删除选区 |
| `Delete` | 删除当前字节 / 删除选区 |
| `Ctrl+C` | 复制选区为十六进制字符串 |
| `Ctrl+V` | 粘贴十六进制字符串到光标位置 |
| `Ctrl+X` | 剪切（复制后删除） |
| `Ctrl+Z` | 撤销（路由到 `editorStore.undo()`，待 EditHistory 集成后生效） |
| `Ctrl+Y` | 重做（同上） |

### 2.3 数据解释器实现（`src/plugins/interpreters/`）

创建了 5 个解释器文件，全部继承 `BaseInterpreter`：

#### `IntegerInterpreter.js`
- `name`: `'integer'`，`displayName`: `'整数'`
- `isFixedLength: true`，`length: 8`
- 支持大小端，按可用字节数依次显示：
  - Int8 / UInt8（1字节）
  - Int16 / UInt16（2字节）
  - Int32 / UInt32（4字节，附带十六进制表示）
  - BigInt64 / BigUint64（8字节，使用原生 `BigInt` API）

#### `Float32Interpreter.js`
- `name`: `'float32'`，`length: 4`
- 使用 `dataView.getFloat32(0, isLittleEndian)`
- 处理 NaN、±Infinity、-0 特殊值
- 根据数量级自动切换定点/科学记数法

#### `Float64Interpreter.js`
- `name`: `'float64'`，`displayName`: `'Float64 (Double)'`，`length: 8`
- 使用 `dataView.getFloat64(0, isLittleEndian)`
- 同 Float32 的特殊值处理

#### `AsciiInterpreter.js`
- `name`: `'ascii'`，`length: 1`
- 0x20-0x7E：显示字符及十六进制，如 `'A'  (0x41)`
- 0x00-0x1F：显示转义名称，如 `\n (LF)`、`\t (TAB)`
- 0x80-0xFF：显示 `—  (非ASCII: 0xXX)`

#### `Utf8Interpreter.js`
- `name`: `'utf8'`，`length: 4`（最长 UTF-8 字符 4 字节）
- 读取首字节判断 UTF-8 序列长度（1/2/3/4 字节）
- 使用 `TextDecoder('utf-8', { fatal: true })` 解码
- 显示字符和 Unicode 码点，如 `'中'  U+4E2D`
- 解码失败或序列不完整时显示错误信息

## 四、架构决策

### 4.1 触发器 ref 模式（而非直接调用）

`editorStore` 的 `copy/paste/cut/commitByte` 使用 ref 触发器而非直接操作文件，原因：
- `editorStore` 不应直接依赖 `fileStore`（单向依赖原则）
- 实际的文件读写由 HexGrid.vue 中的 `watch` 完成，HexGrid 同时持有两个 store 的引用
- 便于后续在不同组件中重用这些操作

### 4.2 半字节输入模式

十六进制编辑器的标准交互：
- 输入第一个字符（高4位）：在光标字节显示 `X_`（待实现高亮）
- 输入第二个字符（低4位）：提交完整字节，光标自动前进
- 切换光标位置时清除未完成输入

### 4.3 剪贴板格式

- **复制输出**：`FF 00 1A 3B`（大写，空格分隔，每字节两位）
- **粘贴解析**：过滤所有非十六进制字符，每两位一字节（兼容空格/逗号/换行等分隔符）
- 使用 `navigator.clipboard` API（需 HTTPS 或 localhost）

## 五、已知问题 / 待优化

1. **输入法高亮缺失**：输入高4位后，光标字节应显示为 `X_` 样式，当前没有视觉反馈，用户不确定是否已输入了半个字节。需在 HexGrid.vue 的 `getByteClass` 中增加 `inputBuffer` 判断。

2. **EditHistory 未集成**：`Ctrl+Z/Y` 的事件路由已实现，但 `editorStore.undo/redo()` 还是空方法，需在下一阶段完成 `WriteCommand`/`InsertCommand`/`DeleteCommand` 并接入 `fileStore`。

3. **DataInfoPanel 中 IntegerInterpreter 显示多行**：`IntegerInterpreter.interpret()` 返回多行字符串（用 `\n` 分隔），但 DataInfoPanel.vue 的 item-value 单元格是单行显示，多行内容会截断。需要将 IntegerInterpreter 拆分为多个独立解释器，或修改面板支持多行。

4. **剪贴板粘贴奇数位问题**：当前实现要求粘贴内容为偶数位十六进制字符（`hexStr.length % 2 !== 0`），若用户粘贴奇数位内容（如 `0F1`）会被直接拒绝，应改为末尾补 `0` 处理。

5. **插入模式下新文件大小**：插入模式每次输入会扩展文件大小，但 `fileStore.activeFile.size` 在 `writeBytes` 中已正确更新，虚拟滚动行数应自动更新，需实测确认。
