## Plan: 字符串解码列功能

**整体思路**：`Decoded` 列改为单字节 ASCII（每行独立解码，不跨行），同时新增一个「字符串解码」列，该列的内容由用户手动指定区域和编码，数据预计算后跨行存储，从根本上解决跨行编码的问题。

---

### 核心设计决策

**为何不直接跨行做 UTF-8 Decoded？**
现有 Decoded 是按行调用 `getRowDecoded(rowIndex)`，每次只能看到本行字节，无法感知前后行。字符串解码列用独立的 Store 预存整个区域的解码结果，渲染时只是查表，可以轻松跨行。

**TextDecoder streaming 逐字节映射**
用 `new TextDecoder(encoding, {fatal: true})` + `decode(byte, {stream: true})` 逐字节喂入：
- 输出空字符串 → 该字节是多字节序列的前导/续接字节，记为占位
- 输出字符 → 该字节是多字节序列最后一个字节，charMap[offset] = char；前面的字节记为 `' '`（空格）
- 抛出异常 → 无效字节，停止

这一 API 是浏览器原生的，支持 utf-8 / utf-16le / utf-16be / gbk(gb18030) / big5 / shift-jis 等所有 `TextDecoder` 支持的编码。

### 实现步骤

**Phase 1 — stringDecodeStore**（*前置，其他步骤依赖它*）
- 数据结构：`{ id, startOffset, endOffset, encoding, charMap: Map<offset, string|null> }`
  - `null` = 超出文件末尾 / 区域外
  - `' '` = 多字节序列占位
  - char = 该字节处显示的字符
- `addRegion(start, end, encoding, fileStore)` 主函数：
  1. 逐字节 streaming 解码，构建 charMap

  4. 中途无效：停止，返回 `{ ok: true, partialEnd: 停止位置 }`
  5. 重叠处理：新区域与已有区域部分重叠时，裁剪旧区域（或删除若完全被覆盖）

### Steps

**Phase A — stringDecodeStore（其他所有步骤前置依赖）**

新建 `src/stores/stringDecodeStore.js`：

- 数据结构 `StringRegion = { id, startOffset, endOffset, encoding, charMap: Map<absOffset, char|' '|'\uFFFD'> }`
  - `' '`（空格）= 多字节序列的前导/续接字节（不显示字符）
  - `char` = 多字节序列最后一个字节处显示的字符  
  - `'\uFFFD'` = 无效字节，显示替代符
- 核心算法 `buildCharMap(startOffset, bytes, encoding, strict)` 使用 `TextDecoder` + `{ stream: true }` 逐字节喂入：
  - 每字节 `decode(chunk, {stream:true})`：返回 `''` → 续接字节，记入 pendingOffsets；返回字符 → 当前 offset 存入 char，pendingOffsets 各存 `' '`
  - `strict=true` (初次创建)：遇异常停止，返回 `{ ok, stopOffset, charMap }`
  - `strict=false` (替换后更新)：遇异常写入 `'\uFFFD'`，继续解码
- `addRegion(start, end, encoding, fileStore)`:
  1. 读取 `[start, end]` 字节
  2. 检查末尾是否有不完整序列：多尝试最多 3 个字节（只读不修改 end，若成功则扩展 endOffset）
  3. 文件结尾无更多字节则截断
  4. 首字节即无效 → 返回 `{ ok: false, reason }`
  5. 中途无效 → 保留有效前段，返回 `{ ok: true, truncated: true, actualEnd }`
  6. 重叠裁剪：新区域与已有区域重叠时，缩短旧区域的 startOffset/endOffset（完全被覆盖则删除）
  7. 成功后 push 进 `regions`
- `updateAffectedRegions(writeOffset, writeLength, fileStore)` —— 替换模式写入后调用：找出与 `[writeOffset, writeOffset+writeLength-1]` 重叠的所有区域，对每个区域重新读取其 `[startOffset, endOffset]` 字节并执行 `buildCharMap(strict=false)` 刷新 charMap（reactive，自动触发视图更新）
- `getCharAtOffset(offset)` → `char | null`（null 表示该 offset 不在任何区域内）
- `getRegionsInRow(rowStart, rowEnd)` → 与该行重叠的区域数组（用于标签条渲染）
- `adjustForDeletion(start, length)` / `adjustForInsertion(offset, length)` —— 与 tagStore 同样的偏移量跟随逻辑，同时对受影响区域重新解码
- `removeRegion(id)`, `clearAll()`

支持的编码列表（`TextDecoder` 原生支持）：
```

```
